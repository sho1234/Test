# ソフトウェア研修

## 目的
*チーム開発において1人でコーディングできるようになること*

```
大規模なソフトにおいて，チーム開発は付き物である．
チーム開発において重要なことは，保守性，可読性，拡張性，再利用性を考えてコーディングすることである．
1人の軽率な行動がソフト，そして，物件全体のQCDを左右する．
```

## 目標
- C#の使い方を学ぶ
- メトロソフトグループのコーディングポリシーを学ぶ
	- 単に「こう書かなければならないから書く」のではなく「なぜそう書かなければならないか」を考える
	- でなければ，ほかの事業部のポリシーをまた暗記する必要がある
	- 高品質なソフトウェアを実現する目的は同じであるのでここで学んだことはほかでも通用する
- デバッグ方法を学ぶ
	- 1人で問題解決する能力
